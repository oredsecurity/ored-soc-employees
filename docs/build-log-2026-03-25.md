### Active Response: firewall-drop on Windows (FAILED)
- Sent firewall-drop command to agent 001 via PUT /active-response
- Wazuh API returned affected_items: 1 (command dispatched)
- Verification on endpoint: no firewall rule created
- Root cause: firewall-drop is a Linux bash script, agent 001 is Windows
- The Wazuh API does not validate OS compatibility. It reports success on dispatch, not execution.
- Lesson: API "success" means "sent to agent," not "executed on endpoint." Always verify on the endpoint.

## What Failed and Why

### 1. Wazuh API Authentication (resolved)
- Dashboard credentials (admin/password) don't work on the API (port 55000)
- Dashboard uses OpenSearch auth, API has its own user store
- Fix: Used wazuh-wui credentials from wazuh.yml on the dashboard node

### 2. Active Response Not Configured (resolved)
- Manager had command definitions but no active-response block in ossec.conf
- API returned error 1652: "The command used is not defined in the configuration"
- Fix: Added active-response block to ossec.conf, restarted manager

### 3. Active Response Command Name Format (resolved)
- Tried firewall-drop0 (0 suffix convention from older docs): failed
- Tried firewall-drop (plain name): failed
- Tried !firewall-drop (! prefix): succeeded
- Wazuh 4.14 uses ! prefix for active response API calls

### 4. Linux Command on Windows Agent (resolved)
- firewall-drop is a bash script, silently fails on Windows
- Wazuh API reports success (command dispatched) regardless of execution
- Fix: Used netsh (Windows-native) instead
- Long-term fix: OS-aware command selection (GitHub Issue #9)

## Wazuh API Notes (for future reference)

- API version: 4.14.4
- Auth endpoint: POST /security/user/authenticate (returns JWT)
- Active response: PUT /active-response?agents_list=XXX
- Command prefix: ! (not 0 suffix)
- API user: wazuh-wui (not admin)
- affected_items means "dispatched," not "executed"
- No /alerts endpoint on this version (alerts in indexer only)
- SCA data available via GET /sca/{agent_id}/checks/{policy_id}
